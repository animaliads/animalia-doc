# <h1 style={{ fontSize: "3rem" }}>Calendar</h1>

<div style={{ maxWidth: '60%' }}>

O componente `Calendar` é usado para selecionar datas próximas ao tempo presente. Por padrão, os seletores de data em calendário apresentam os dados do mês atual e apenas um mês é apresentado por vez, podendo, em alguns contextos, exibir uma data pré-estabelecida para a pessoa usuária. Normalmente mostram os dias da semana na parte superior e combinam com a metáfora de calendários de mesa ou de parede.

  ### Quando usar

  - Para selecionar datas próximas ao tempo presente para que a pessoa usuária não precise navegar tanto.
  - Dentro de um componente [`Datepicker`](/docs/components/datepicker) (acionado pelo clique em um ícone de calendário).
  - Quando a visualização em grade mensal facilita a compreensão do contexto (ex: dias da semana).

  ### Quando não usar

  - **Datas distantes:** Não use para solicitar datas no passado ou futuro distante, como data de nascimento. Nesses casos, use um campo de texto para evitar excesso de cliques.
  - **Seleção de horários:** O calendário é estritamente para datas; para horários, utilize componentes de tempo (Timepicker).

  ### Anatomia

  O calendário diferencia visualmente o dia atual, o dia selecionado e o dia em estado de hover. Essa distinção não se apoia apenas em cores, garantindo a acessibilidade.

  :::

  ### Comportamento e Responsividade

  - **Visibilidade:** Mantenha sempre visível qual mês e ano a pessoa usuária está visualizando.
  - **Datas Ilógicas:** Deve-se impedir a seleção de datas que não fazem sentido no contexto (ex: data de retorno anterior à data de partida).
  - **Datas Inativas:** Datas indisponíveis (como feriados ou datas passadas em fluxos de agendamento) devem ser mantidas inativas por meio de opacidade e ter o clique desabilitado.

  ## Propriedades Customizáveis (Tokens)

  O componente possui os estados: **Default (Normal)**, **Hover**, **Focus**, **Selected (Selecionado)** e **Disabled (Inativo)**.

  ### Calendar (Geral)

  Propriedades de base que definem a estrutura global do componente.

  | **Propriedade** | **Descrição** | **Valor padrão** |
  | --- | --- | --- |
  | `--font-family` | Família tipográfica usada em todo o componente | `$font-family-text` |
  | `--background-color` | Cor de fundo da base do calendário | `$color-neutral-light-00` |
  | `--border-color` | Cor da borda externa do componente | `$color-neutral-mid-40` |
  | `--border-width` | Espessura da borda da base | `$border-width-sm`  |
  | `--border-radius` | Arredondamento dos cantos do componente | `$border-radius-md` |
  | `--shadow` | Sombra aplicada à base (elevação) | `$shadow-level-1` |

  ### Header (Navegação)

  Tokens para os seletores de mês/ano e botões de navegação lateral.

  | **Propriedade** | **Descrição** | **Valor padrão** |
  | --- | --- | --- |
  | `--header-text-color` | Cor do texto do mês e ano no cabeçalho | `$color-neutral-dark-90` |
  | `--header-icon-color` | Cor dos ícones de seta (anterior/próximo) | `$color-action-default` |
  | `--header-padding` | Espaçamento interno do cabeçalho | `$spacing-sm` |
  | `--header-gap` | Espaço entre os seletores e as setas | `$spacing-xs` |

  ### Grade de Dias (Grid)

  Propriedades aplicadas aos nomes dos dias da semana e às células de data.

  | **Propriedade** | **Descrição** | **Valor padrão** |
  | --- | --- | --- |
  | `--day-name-color` | Cor do texto dos dias da semana (ex: dom, seg) | `$color-neutral-mid-60` |
  | `--day-text-color` | Cor padrão do número do dia | `$color-neutral-dark-90` |
  | `--day-font-size` | Tamanho da fonte dos números | `$font-size-sm` |
  | `--day-background-hover` | Cor de fundo da célula ao passar o mouse | `$color-brand-01-lighter` |
  | `--day-background-selected` | Cor de fundo do dia selecionado | `$color-action-default` |
  | `--day-text-selected` | Cor do texto do dia selecionado | `$color-neutral-light-00` |
  | `--day-color-today` | Cor de destaque para o dia atual | `$color-action-default`  |
  | `--day-color-disabled` | Cor do texto para datas inativas/ilógicas | `$color-neutral-mid-40`  |

  ### Footer (Ações)

  Tokens para os botões de ação rápida localizados na base do componente.

  | **Propriedade** | **Descrição** | **Valor padrão** |
  | --- | --- | --- |
  | `--footer-padding` | Espaçamento interno da área de rodapé | `$spacing-sm` |
  | `--footer-button-color` | Cor do texto dos botões "Limpar" e "Hoje" | `$color-action-default` |
  | `--footer-border-top` | Linha divisória entre a grade e o rodapé | `$border-width-sm` |

  ### Como utilizar

  A implementação requer a importação do módulo principal, variando conforme a biblioteca ou framework utilizado. Após a configuração, utilize o elemento `ani-calendar` no seu código:

  ```html
  <ani-calendar></ani-calendar>
  ```

  >Para aplicar o tema padrão, é obrigatória a instalação e o uso dos pacotes `@animaliads/animalia-brand` e `@animaliads/css-tokens`. Você pode conferir exemplos práticos no [Live Demo](#live-demo).
  

  ### Customização

  Para adaptar visualmente o componente, você pode sobrescrever as propriedades customizáveis via CSS:

  ```css
  ani-calendar {
    --color-selected: #7000FF;
    --font-family: 'Roboto', sans-serif;
    --opacity-disabled: 0.4;
  }
  ```

  >Para saber quais são as propriedades disponíveis, consulte a aba de [Estilização](/docs/components/calendar#propriedades-customizáveis-tokens).

  ## Acessibilidade (Padrão AAA)
  
  As diretrizes abaixo foram implementadas para garantir que o componente seja compreensível e operável por todas as pessoas, conforme os critérios da [WCAG 2.1](https://www.w3.org/TR/WCAG21/).

  ### Interação por Teclado

  A navegação deve ser lógica e permitir o controle total sem o uso de mouse:

  | **Tecla** | **Função** |
  | --- | --- |
  | `Up Arrow` / `Down Arrow` | Movem o foco para o mesmo dia da semana anterior ou da próxima semana. |
  | `Left Arrow` / `Right Arrow` | Movem o foco para o dia anterior ou para o dia seguinte. |
  | `Home` / `End` | Movem o foco para o primeiro ou o último dia da semana atual. |
  | `Page Up` / `Page Down` | Alteram a visualização para o mês anterior ou para o próximo mês. |
  | `Shift + Page Up` / `Shift + Page Down` | Alteram a visualização para o ano anterior ou para o próximo ano. |
  | `Tab` | Percorre sequencialmente as partes interativas (Setas de navegação, Seletores e Grade). |
  | `Esc` | Fecha o componente sem realizar alterações indesejadas. |

  ### Operável e Compreensível

  - **Navegação Lógica:** Ao usar a tecla `Tab`, o foco percorre o calendário em uma sequência intuitiva: começa pelos controles de navegação e seletores de mês e ano no topo (**Header**), avança para a grade de seleção de datas (**Day Grid**) e termina nos botões de ação rápida na parte inferior (**Footer**). ([WCAG 2.4.3](https://www.w3.org/WAI/WCAG22/Understanding/focus-order.html)).
  - **Leitura por Voz:** O componente utiliza etiquetas internas que explicam para quem usa leitores de tela qual a função de cada botão e qual o estado de cada data (por exemplo: se o dia está disponível, se já foi selecionado ou se é o dia atual). ([WCAG 4.1.2](https://www.w3.org/WAI/WCAG22/Understanding/focus-visible.html))
  - **Ações Previsíveis:** Navegar ou colocar o foco sobre uma data não causa mudanças bruscas na página (como enviar um formulário sozinho). As ações só acontecem quando a pessoa usuária confirma a escolha. ([WCAG 3.2.1](https://www.w3.org/WAI/WCAG21/Understanding/on-focus.html))
