## Visão geral

Os botões são utilizados para realizar ações e para tomada de decisões. Podem ser utilizado em páginas e também dentro de outros componentes.

### Boas práticas

- Use labels (rótulos) diretos, curtos e intuitivos, que direcionem o usuário para o que será realizado. Para isso, prefira utilizar verbos, por exemplo: "Salvar" e "Adicionar";
- A interação de foco deve compor os botões, sendo navegação por mouse ou teclado, e em sequência lógica;
- Evite utilizar muitos botões em uma única tela, pois isso aumenta a carga cognitiva do usuário e vai aumentar seu tempo de decisão;
- Não use cores para label e backgroud com baixo contraste entre elas, pois isso dificulta o entendimento do conteúdo do botão.

### Anatomia

## Acessibilidade

O componente botão foi projetado para atender os requisitos das Diretrizes de Acessibilidade para Conteúdo Web (WCAG) 2.1, mas alterações feitas na aplicação deste podem afetar essa conformidade.

Por isso, é muito importante que, ao aplicar esse componente, o proprietário do conteúdo leve em consideração os seguintes critérios:

**Operável**

- O botão deve sempre ter um rótulo que descreva precisamente a ação que o mesmo executa. [2.4.6: Headings and Labels](https://www.w3.org/WAI/WCAG21/Understanding/headings-and-labels)
- O nome acessível do botão deve conter o rótulo visível do mesmo. Em caso de descrição adicional, o rótulo deverá ser lido primeiro. [2.5.3: Label in Name](https://www.w3.org/WAI/WCAG21/Understanding/label-in-name)
- O estado de foco deve ser sempre preservado, inclusive quando o botão estiver desabilitado. [2.4.7: Focus Visible](https://www.w3.org/WAI/WCAG21/Understanding/focus-visible)
- Após a ativação do botão o foco deve ser movido dependendo do tipo de ação que o botão executa. Consultar lista de exemplos em [W3C WAI-ARIA 3.5 Button](https://www.w3.org/TR/wai-aria-practices-1.1/#button)

**Perceptível**

- Entre as cores do rótulo do botão e seu background o contraste deve ser de no mínimo 7:1, para botões ativos. [1.4.6: Contrast](https://www.w3.org/WAI/WCAG21/Understanding/contrast-enhanced.html)

### Diretrizes tratadas no componente

Algumas diretrizes de acessibilidade já são tratadas no componente, internamente, e não podem ser alteradas pelo proprietário do conteúdo. São elas:

- Quando em foco, o botão é ativado usando as teclas de _Espaço_ e _Enter_ do teclado. [W3C WAI-ARIA 3.5 Button - Keyboard Interaction](https://www.w3.org/TR/wai-aria-practices-1.1/#keyboard-interaction-3)

## Desenvolvimento

### Instalação

Você pode instalar o componente de forma isolada, conforme o exemplo abaixo:

```
npm install @animaliads/ani-button
```

Ou instalar a biblioteca inteira de componentes:

```
npm install @animaliads/web-components
```

### Como utilizar

Para utilizar o web component na sua aplicação você precisa importar o arquivo principal:

```html title="html"
<script src="./node_modules/@animaliads/ani-button"></script>
```

Mas caso você estiver utilizando algum framework, pode ser necessário seguir a documentação do próprio
componente para adicionar o web component. Ou apenas importar o arquivo do componente
em um arquivo JS, por exemplo:

```javascript title="javascript"
import '@animaliads/ani-button';
```

Depois dessa configuração, basta adicionar a tag `ani-button` no seu código:

```html title="html"
<ani-button>Hello world!</ani-button>
```

Para customizar o tema do componente, você pode estar utilizando as propriedades customizáveis
da seguinte forma:

```css title="css"
ani-button {
  --background: red;
  --line-height: 100%;
  --border-radius: 3px;
}
```

Para saber quais são as propriedades disponível, consulte [propriedades customizáveis do botão](#propriedades-customizáveis).

## Propriedades customizáveis

O botão possui propriedades customizáveis que podem variar de acordo com o tema selecionado. As propriedades customizáveis podem ser alteradas para cada produto, de acordo com os tokens de marca e tokens globais. Com isso, garantimos uma consistência no Design System, sem perder a identidade de cada marca.

### Button

| Propriedade        | Contexto   |
| ------------------ | ---------- |
| `--font-family`    | Label      |
| `--text-color`     | Label      |
| `--font-size`      | Label      |
| `--line-height`    | Label      |
| `--background`     | Background |
| `--border-radius`  | Background |
| `--spacing-squish` | Background |
| `--shadow`         | Background |

### Propriedades customizáveis em cada estado

| Estado             | Propriedade                | Contexto   |
| ------------------ | -------------------------- | ---------- |
| hover              | `--background-hover`       | Background |
| focus              | `--outline-color-focused`  | Outline    |
| pressed            | `--background-pressed`     | Background |
| pressed            | `--outline-color-pressed`  | Outline    |
| disabled           | `--background-disabled`    | Background |
| disabled           | `--text-color-disabled`    | Label      |
| focusable disabled | `--outline-color-disabled` | Outline    |
