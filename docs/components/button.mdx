  ## **Comportamento e Responsividade**

  ### Variantes

  Cada tipo de botão representa uma prioridade na interface:

  <img 
  src={buttonTypes} 
  alt="Os cinco tipos de botões dispostos lado a lado: Primary, Secondary, Tertiary, Primary Danger e Secondary Danger" 
  style={{ 
    display: 'block',      
    margin: '0 auto',      
    maxWidth: '50%',       
    marginBottom: '2rem',  
    backgroundColor: '#f9f9fa',
    padding: '1rem',
    borderRadius: '8px'  
  }} 
  />

  - **Primary:** Usado para a ação principal da página (preferencialmente apenas um por página).
  - **Secondary:** Usado para ações secundárias, ao lado de botões primários ou isolados.
  - **Tertiary:** Para ações menos importantes; se isolado, use com ícone para facilitar o entendimento.
  - **Primary Danger:** Para ações principais de grande impacto e difícil reversão (ex: excluir conta). Recomenda-se ícone de alerta junto à label.
  - **Secondary Danger:** Para a segunda ação de grande impacto (ex: cancelar processos importantes).

  ### Tamanhos e Largura

  - **Escalabilidade:** Disponível em **Small**, **Medium (padrão)** e **Large**.

  Caso você já tenha feito o passo-a-passo descrito em [Começando](/docs/about/getting-started), você pode pular esta instalação e ir direto para o tópico [Como utilizar](#como-utilizar).
  Você pode instalar o componente de forma isolada, conforme o exemplo abaixo:

  ```
  npm install @animaliads/ani-breadcrumb
  ```

  ### Como utilizar

  A implementação requer a importação do módulo principal, variando conforme a biblioteca ou framework utilizado. Após a configuração, utilize o elemento `ani-button` no seu código:

  ```html
  <ani-button>Salvar cadastro</ani-button>
  ```
  >Para aplicar o tema padrão, é obrigatória a instalação e o uso dos pacotes  `@animaliads/animalia-brand` e `@animaliads/css-tokens`. Você pode conferir exemplos práticos no [Live Demo](#live-demo).

  ### Customização
  Para adaptar visualmente o componente, você pode sobrescrever as propriedades customizáveis via CSS:

  ```css
  ani-button[kind="secondary"]{
    --border-color: purple;
    --line-height: 100%;
    --border-radius: 3px;
  }
  ```
  >Para saber quais são as propriedades disponíveis, consulte [propriedades customizáveis do button](/docs/components/button#propriedades-customizáveis).

  ## Acessibilidade (Padrão AAA)

  As seguintes diretrizes são tratadas internamente pelo componente:

  - **Interação de Teclado:** Ativação via teclas `Espaço` e `Enter` quando em foco.
  - **Aparência do Foco:** Espessura mínima de **2 pixels** CSS, visível e nunca escondido por outros elementos.
  - **Preservação de Estado:** O estado de foco é sempre preservado, inclusive quando o botão estiver desabilitado.
  - **Nome Acessível:** O nome lido por leitores de tela deve ser o mesmo nome definido na label visual ([WCAG 2.5.3](https://www.w3.org/WAI/WCAG22/Understanding/label-in-name.html)).
  - **Contraste (WCAG 2.1):**
      - **Nível AA (Mínimo):** Relação de contraste de pelo menos **4.5:1** entre o texto e o fundo.
      - **Nível AAA (Excelência):** Relação de contraste de pelo menos **7:1** entre o texto e o fundo.
