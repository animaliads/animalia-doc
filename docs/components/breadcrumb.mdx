# <h1 style={{ fontSize: "3rem" }}>Breadcrumb</h1>

<div style={{ maxWidth: '60%' }}>

O **Breadcrumb** (trilha de navegação) é um componente secundário que auxilia a pessoa usuária a compreender sua localização atual na hierarquia de um site ou sistema.
Ele permite o retorno rápido a níveis superiores através de links clicáveis que representam o caminho percorrido.

### Quando usar

* Essencial em sistemas com arquitetura de informação profunda, possuindo mais de 2 níveis.
* Quando a localização do usuário não é óbvia apenas pelo menu de navegação principal.

### Quando não usar

* Em fluxos lineares e sequenciais, como processos de compra ou cadastros passo a passo (utilize o componente Stepper).
* Em sites ou aplicações com estruturas simples de apenas um nível.

### Comportamento e Responsividade

* **Navegação**: Cada nível clicável deve levar à respectiva página pai.
* **Responsividade (Single Line)**: O componente deve ocupar sempre uma única linha.
    * No **Desktop**, exibe-se a trilha completa.
    * No **Mobile**, os níveis intermediários são colapsados automaticamente no ícone (`...`) mantendo visíveis apenas o nível raiz e a página atual.
* **Truncamento**: Títulos longos que excedam o espaço disponível devem ser truncados com reticências. O nome completo deve ser revelado via tooltip no hover ou foco.
* **Interação**:
    * **Hover**: Feedback visual imediato nos links interativos.
    * **Teclado**: Navegação via `Tab` e acionamento por `Enter` / `Space`.
* **Touch**: Área mínima de toque de **1x1cm** (aprox. 44px).

## Desenvolvimento

Caso você já tenha feito o passo-a-passo descrito em [Começando](/docs/about/getting-started), você pode pular esta instalação e ir direto para o tópico [Como utilizar](#como-utilizar).
Você pode instalar o componente de forma isolada, conforme o exemplo abaixo:

```
npm install @animaliads/ani-breadcrumb
```

### Como utilizar

A implementação requer a importação do módulo principal, variando conforme a biblioteca ou framework utilizado. Após a configuração, utilize os elementos `ani-breadcrumb` e `ani-breadcrumb-item` no seu código:

```html
<ani-breadcrumb>
  <ani-breadcrumb-item href="/">Home</ani-breadcrumb-item>
  <ani-breadcrumb-item href="/produtos">Produtos</ani-breadcrumb-item>
  <ani-breadcrumb-item>Antivírus</ani-breadcrumb-item>
</ani-breadcrumb>
```
>Para aplicar o tema padrão, é obrigatória a instalação e o uso dos pacotes  `@animaliads/animalia-brand` e `@animaliads/css-tokens`. Você pode conferir exemplos práticos no [Live Demo](#live-demo).

### Customização
Para adaptar visualmente o componente, você pode sobrescrever as propriedades customizáveis via CSS:

```css
ani-breadcrumb {
  --color: #4545A1;
  --font-size: 0.875rem;
}
```
>Para saber quais são as propriedades disponíveis, consulte [propriedades customizáveis do breadcrumb](/docs/components/breadcrumb#propriedades-customizáveis).

## Acessibilidade (Padrão AAA)

* **Identificação:** O componente deve estar contido em uma tag `<nav>` identificada com o atributo `aria-label="Breadcrumb"`.
* **Página Atual:** O item estático final (**Current page**) deve possuir o atributo `aria-current="page"`.
* **Separadores:** Devem ser decorativos e ignorados por leitores de tela através do uso de `aria-hidden="true"`.
* **Contraste (WCAG 2.1):** 
    * **Nível AA (Mínimo):** Relação de contraste de pelo menos 4.5:1 entre o texto/link e o fundo.
    * **Nível AAA (Excelência):** Relação de contraste de pelo menos 7:1 entre o texto/link e o fundo.
* **Ordem de Foco:** O foco via teclado deve percorrer apenas os links ativos e o menu de colapso, ignorando a **Current page** para otimizar a navegação.
