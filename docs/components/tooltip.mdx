## Visão geral

O componente `tooltip`  tem a função de apresentar ajuda contextual ou informação adicional sobre um elemento em tela.

## **Boas práticas**

A informação devem ser tão concisas e claras quanto possível. 

### Faça

- Ajudar as pessoas usuárias a entender o significado ou propósito do elementos em tela;
- Fornecer informações adicionais para a conclusão de uma tarefa;
- Fornecer conteúdo breve e útil.

### Não faça

- Informações importantes ou cruciais não devem sempre dentro do tooltip;
- Não utilize o tootltip para passar uma mensagem de erro
- Não obstrua informações importantes
- Não coloque ações dentro de uma tooltip

### **Acessibilidade tratada no componente**

Algumas diretrizes de acessibilidade já são tratadas no componente, internamente, e não podem ser alteradas pelo proprietário do conteúdo. São elas:

- Usar medidas relativas para possibilitar um zoom de até 200%, sem perda do conteúdo ou funcionalidade.([WCAG 1.4.4: Resize Text](https://www.w3.org/WAI/WCAG21/Understanding/resize-text));
- Permitir que o usuário feche o tooltip. ([WCAG 2.2.4: Interruptions](https://www.w3.org/WAI/WCAG21/Understanding/interruptions.html));
- Permitir a interação via teclado, através das teclas: Tab, Esc e setas. ([WCAG 2.1.1: Keyboard](https://www.w3.org/WAI/WCAG21/Understanding/keyboard));
- A ordem de foco deve acontecer em sequência lógica. ([WCAG 2.4.3: Focus Order](https://www.w3.org/WAI/WCAG21/Understanding/focus-order))
- Utilizar nome, função e valor adequadamente, pois estes parâmetros são utilizados por tecnologias assistivas para identificar os elementos da interface. ([WCAG 4.1.2: Name, Role, Value](https://www.w3.org/WAI/WCAG21/Understanding/name-role-value)).
